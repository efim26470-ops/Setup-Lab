# SetupLab PWA 1.5.0

SetupLab — статическое PWA для подбора, сравнения и локального учёта конфигураций ПК, автосима, домашнего кинотеатра, рабочего места, фотооборудования и музыкальных систем. Проект рассчитан на GitHub Pages: сервер и аккаунт не требуются.

## Мега-каталог

В версии 1.5.0 доступно **2900 позиций**:

- ПК — **1000**;
- автосим — **500**;
- домашний кинотеатр — **350**;
- рабочее место — **350**;
- фотооборудование — **350**;
- музыкальная система — **350**.

Распределение ПК:

- 120 процессоров;
- 140 видеокарт;
- 150 материнских плат;
- 140 комплектов памяти;
- 140 накопителей;
- 100 блоков питания;
- 110 корпусов;
- 100 систем охлаждения.

Распределение автосима:

- 70 рулевых баз и готовых комплектов;
- 90 рулей;
- 70 педальных узлов;
- 70 кокпитов и стоек;
- 40 шифтеров;
- 35 ручных тормозов;
- 35 телеметрических панелей;
- 90 креплений, адаптеров и аксессуаров.

В каталоге есть бюджетные, средние и флагманские варианты. Для рабочего места добавлено 100 мониторов, включая недорогие FHD-модели, 1440p, ультраширокие и профессиональные дисплеи.

Каждая позиция содержит описание, характеристики, ориентировочную цену, баллы, апгрейдность и ссылки на поиск в Яндекс Маркете, Ozon, DNS для поддерживаемых категорий и официальный источник.

## Важное замечание о данных

Расширенный каталог объединяет отдельно подготовленные модели и параметризованные варианты реальных товарных семейств. Для таких вариантов цена и конкретный индекс являются ориентиром: перед покупкой необходимо проверить точное название, комплектацию, наличие и совместимость по ссылкам из карточки. SetupLab не помечает эти значения как цены в реальном времени.

## Строгий мастер подбора

Мастер теперь учитывает **полную стоимость всей сборки**, а не подбирает компоненты только по баллам.

Пользователь задаёт:

- направление;
- общий бюджет в выбранной валюте;
- приоритет — баланс, производительность, цена/возможности или апгрейдность;
- предпочтительный бренд;
- страну покупки;
- для автосима — платформу, дисциплину и способ установки.

Создаются три сценария с отдельными жёсткими лимитами:

- экономный — до 65% указанного бюджета;
- сбалансированный — до 85%;
- максимальный — до 100%.

Алгоритм:

1. резервирует деньги на обязательные компоненты;
2. сначала обеспечивает полноту и совместимость;
3. учитывает интегрированные автосим-комплекты с рулём и педалями;
4. проверяет платформу, экосистему, сокеты, память, питание, размеры и другие ограничения;
5. улучшает комплект только пока итоговая сумма остаётся в лимите;
6. показывает использованный процент бюджета и остаток;
7. блокирует сохранение варианта, если даже самый дешёвый совместимый комплект дороже лимита.

## Публикация на GitHub Pages

1. Полностью замените содержимое репозитория файлами из архива.
2. Откройте **Settings → Pages**.
3. Выберите **Deploy from a branch**.
4. Укажите ветку `main` и папку `/ (root)`.
5. Дождитесь завершения публикации.
6. На iPhone закройте старую вкладку SetupLab и откройте сайт повторно.

Если Safari показывает старую версию, один раз откройте сайт с параметром `?v=1.5.0` либо используйте кнопку локального восстановления.

## Файлы проекта

- `index.html` — оболочка интерфейса;
- `styles.css` — адаптивный дизайн и темы;
- `app.js` — каталог, сборки, расчёты и PWA;
- `features.js` — мастер, Sim Racing Lab и инженерная аналитика;
- `catalog.json` — полный редактируемый каталог;
- `catalog.min.json` — компактная копия для запуска;
- `sw.js` — офлайн-кэш;
- `manifest.webmanifest` — PWA-манифест;
- `CATALOG_NOTES.md` — методика расширенного каталога.

## Ограничения статического проекта

GitHub Pages не выполняет серверный парсинг магазинов. Цены остаются редактируемыми ориентирами, а история накапливается локально. Перед покупкой необходимо проверить точную модификацию, комплектацию, гарантию, совместимость, стоимость доставки и наличие у продавца.
